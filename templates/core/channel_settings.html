Browser Sessions & Cookie Integration

Load active logged-in browser cookies to bypass login walls and multi-factor authentication (MFA) checks during automated job applications.

Setup Instructions
  1. Login to your target job portals (e.g., LinkedIn, Naukri, Internshala) on your primary Chrome/Firefox browser.
  2. Use a free browser extension like EditThisCookie or open the developer console (F12) on the portal tab and run this copy-paste script: 
    copy(JSON.stringify(document.cookie.split("; ").map(c => {
    const [name, ...val] = c.split("=");
    return {
        name: name,
        value: val.join("="),
        domain: window.location.hostname.startsWith(".") ? window.location.hostname : "." + window.location.hostname,
        path: "/"
    };
})));
  3. Save the copied JSON into a file named after the portal (e.g. linkedin.json, naukri.json, or internshala.json).
  4. Place the file in your project directory at:
    media/browser_sessions/cookies/{portal_name}.json
💡
Pro-Tip: Playwright will automatically load these cookies when initiating stealth runs, allowing the agent to fill forms as if you are browsing directly.
